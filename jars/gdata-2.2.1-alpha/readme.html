Java Client Library for Google API's version 2.2.1-alpha

This version 2 library is packaged in two different ways:

  • Android (gdata-android-2.2.1-alpha.zip)
  • (this download) All other Java 5 environments, including App Engine (gdata-java-2.2.1-alpha.zip)

See also the JavaDoc.

If you encounter a problem with this client library or want to request a feature, please submit an issue.

For discussions related to this client library only, please visit GData Java Client Group.

WARNING: this is an experimental branch of the library, and as such the client library API is subject to change. There are known bugs, and important features not yet implemented. We invite you to experiment with this library and provide us with early feedback.

Library Jars

When building an application for all Java environments other than Android, simplify your build process by just using the all-in-one jar.

  • gdata-2.2.1-alpha.jar: required
  • dependencies/jackson-core-asl-1.5.0.jar: needed for user-defined JSON data models
  • dependencies/kxml2-2.3.0.jar: needed for user-defined XML data models
  • dependencies/commons-logging-1.1.1.jar, dependencies/httpcore-4.0.1.jar, and dependencies/httpclient-4.0.1.jar: needed only if you prefer to use the Apache HTTP library for the underlying HTTP transport (default is to use the built-in java.net package of the JDK). Warning: Do not use the Apache HTTP library on App Engine because it won't work. It tries to open its own sockets which is not allowed on App Engine.
